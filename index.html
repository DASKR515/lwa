// 01 — the problem

Building Lua on Windows is Broken.

Most developers spend hours fighting compilers before writing a single line of Lua.

The Old Way

Install TCC, fight linker errors.
Try MinGW, get DLL mismatches.
Try Clang, discover missing headers.

Spend days or weeks just trying to install one LuaRocks package. Most developers give up before they even start.

The LWA Way

After months of stress-testing every compiler on Windows, one truth emerged: cl.exe (Visual Studio) causes the fewest problems.

LWA pre-configures the entire toolchain. Install once, write Lua immediately. No PATH editing. No compiler hunting. No broken builds.

// 02 — how it works

Three Steps to Ready.

From zero to a working Lua environment in under 2 minutes.

01

Download & Install

Run the .exe installer. LWA sets up Lua 5.4, LuaRocks, all core libraries, and IUP in one shot. No manual configuration needed.

02

Activate with run.ps1

Open PowerShell and run run.ps1. This activates the isolated LWA environment — your PATH, LUA_PATH, and LuaRocks config are all set automatically.

03

Write Lua

Install any library with luarocks install. Packages stay inside your project — your system stays clean. Build GUIs with IUP. Ship your app.

// 05 — get started

Start in 60 Seconds.

Download the installer or clone the repo and run the environment manually.

1. Download and run the installer below
2. Launch PowerShell in your project folder
3. Run run.ps1 to activate the environment
Download lwa-setup.exe
PS> git clone https://github.com/DASKR515/lwa
PS> cd lwa
PS> .\run.ps1
> -- After activating run.ps1
PS> luarocks install inspect
PS> lua main.lua
Libraries install inside your project — not globally.
